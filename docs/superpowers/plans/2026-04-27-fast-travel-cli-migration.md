# fast-travel-cli migration — implementation plan

**Goal:** Migrate `fast-travel-cli` from its sibling repo (`onceuponaprince/fast-travel-cli`) into the ghostroute monorepo as a peer to `ask-grok-cli/` and `ask-perplexity-cli/`. Harmonise `chromiumoxide` to `0.9.1` across all three Rust CLIs as part of the same migration so the integrated repo presents a single dependency baseline.

**Spec:** [`../specs/2026-04-27-fast-travel-cli-design.md`](../specs/2026-04-27-fast-travel-cli-design.md)

**Architecture:** Subtree merge preserves the 5 original fast-travel-cli commits inside ghostroute's history. Old repo is archived (not deleted) on GitHub with a redirect README so external links remain navigable. No Cargo workspace at ghostroute root — each Rust CLI stays an independent crate with its own `Cargo.toml` / `Cargo.lock`.

**Tech stack:** `git subtree`, `cargo check` (verification only — no source-code refactor), `gh repo archive`, shell.

**Operational note:** Steps progress in atomic-commit order. Wind-down (old repo archive + local delete) is *gated on user verify* — the plan pauses after commit 5 and waits for confirmation that ghostroute build works end-to-end before touching the old repo.

## Out of scope

- **Gemini *write* surface (`ask-gemini-cli`).** Symmetric to `ask-grok-cli` / `ask-perplexity-cli` but a separate build. Defer until a real prompt-write use case lands.
- **Range selection, output-format flags, typed errors, tests, installed-binary packaging.** All listed in the spec's *Future work* section. Migration preserves first-run shape, doesn't extend it.
- **Selector-drift fixes.** If Gemini's UI has drifted between 2026-04-26 (last fast-travel-cli commit) and migration day, `dump_diagnostics()` will surface it on first run. Fix as a separate commit if needed.
- **chromiumoxide source-code refactor for the older CLIs.** The 0.7 → 0.9 jump only required dropping a feature flag, not API changes. If a future bump introduces breakage, that's its own commit.
